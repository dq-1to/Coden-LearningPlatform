# Coden - Future Roadmap & Improvement Plan

## 🛠️ 現状の修正・改善案 (Improvements)

現状のアプリ（v1.0）における課題に対する修正案です。

| 項目 | 重要度 | 内容 | 実装案 |
|---|---|---|---|
| **モバイルUXの最適化** | **High** | スマホでの学習体験向上（特にエディタ操作は困難なため、閲覧専用モードへの切り替えなど） | - モバイル時はMonaco EditorをReadOnlyにする<br>- 縦並びレイアウトの調整 |
| **コンテンツ分離** | **High** | `steps.ts` 等のハードコード解消。非エンジニアでも編集可能に | - **Phase 1**: Markdownファイル化 (`src/content/steps/*.md`)<br>- **Phase 2**: Headless CMS (MicroCMS) 導入 |
| **テスト自動化** | **Medium** | アプリ自体の品質保証。リファクタリング時のバグ混入防止 | - **Unit Test**: Vitestでユーティリティ/フックをテスト<br>- **E2E Test**: Playwrightで主要フロー（学習→完了）をテスト |
| **ユーザビリティ** | **Medium** | 初学者向けの用語集、UIガイドの追加 | - ツールチップで専門用語（Props, State等）を解説<br>- 初回起動時のオンボーディングツアー (driver.js等) |
| **パフォーマンス** | **Low** | 初期ロード時間の短縮 | - Monaco Editorの遅延読み込み (React.lazy)<br>- ルートベースのコード分割 |

---

## 🚀 今後の発展機能案 (New Features)

アプリの教育効果を高め、モダンな技術スタックに対応するための新機能案です。

| 機能名 | 重要度 | 概要 | 技術スタック・実装イメージ |
|---|---|---|---|
| **1. Next.js コース** | **High** | Reactの次はNext.js。サーバーサイドコンポーネント(RSC)やルーティングを学ぶ | - `json-server` ではなく本物のNext.js API Routesを使用<br>- Vercelへのデプロイ体験 |
| **2. TypeScript 道場** | **High** | 型定義に特化したパズル・クイズ。Monaco Editorの型チェック機能を活用 | - `Pick`, `Omit`, `Generic` の穴埋め<br>- コンパイルエラーを解消するゲームモード |
| **3. AIチューター** | **Medium** | 学習中のコードをAIが解析し、リアルタイムでヒントや解説を提供 | - Gemini API等のLLM統合<br>- 「なぜエラーが出ているの？」ボタンの実装 |
| **4. ユーザー投稿機能** | **Medium** | ユーザー自身が問題を作成・共有できるプラットフォーム化 (UGC) | - Supabase / Firebase Auth (認証)<br>- Firestore (問題データ保存)<br>- 「いいね」ランキング |
| **5. チーム開発体験** | **Low** | 疑似的なチーム開発。Git操作やコードレビューを体験 | - 仮想Gitクライアント実装<br>- 「PRを作成してマージする」フローのシミュレーション |

---

## 📅 推奨スケジュール

### Step 1: 足場固め（1-2週間）
まずは **モバイルUX最適化** と **コンテンツ分離（Markdown化）** を実施し、学習しやすさと運用性を向上させる。

### Step 2: 機能拡張（2-4週間）
**TypeScript 道場** を追加。現在のMonaco Editor基盤をそのまま活かせるため、コスト対効果が高い。

### Step 3: モダン化（1ヶ月〜）
**Next.js コース** の追加。アプリ自体のフレームワークをNext.jsに移行することも視野に入れる（または別リポジトリとして作成）。
